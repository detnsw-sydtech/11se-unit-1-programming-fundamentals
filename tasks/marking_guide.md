# Marking Guide — Interactive Calculator

## Criteria

### 1. Functionality (10 marks)
- Correct input handling (2)
- Correct operation selection (2)
- Correct calculations (3)
- Clear output formatting (3)

### 2. Code Quality (10 marks)
- Meaningful variable names (2)
- Appropriate use of functions (3)
- Readable structure and formatting (3)
- Comments where needed (2)

### 3. Error Handling (5 marks)
- Non‑numeric input handled (2)
- Invalid operation handled (1)
- Division by zero handled (2)

### 4. Testing Evidence (5 marks)
- Demonstrated testing with varied inputs (3)
- Logical debugging approach (2)

### Total: **30 marks**

---

## Teacher Notes
- Look for incremental development and clear commit history  
- CodeQL alerts should not be used as marks, but may guide feedback  
- Encourage students to refine their code based on early testing
